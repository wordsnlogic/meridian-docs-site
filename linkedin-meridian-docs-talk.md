# LinkedIn post draft for Meridian Docs talk

## Main post

Headline:
A docs team hits a real-world bottleneck and docs as code fixes it

Post copy:

> In this talk I tell the story of a documentation team that started simple, then hit the classic scaling trap: content spread across drafts, review became messy, and every update felt risky.
> 
> I walk you through that practical struggle in a narrative format, the part where the team spends too much time fixing formatting, chasing feedback, and wondering if the final published docs will actually match what reviewers saw.
> 
> Then I show how docs as code turns that story around by introducing:
> - Git + Markdown for clear version history
> - Docusaurus for structured docs, API reference, and release notes
> - GitHub Actions + Netlify for preview URLs and safer deploys
> 
> The result is not just better tools. It is a workflow that helps docs teams feel confident, avoid manual rework, and keep content aligned with product change.
